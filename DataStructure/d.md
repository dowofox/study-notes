📚 자료구조 (Data Structure)
📌 개념

컴퓨터에서 처리할 데이터를 효율적으로 관리하고 구조화하기 위한 방법

📂 자료구조의 분류
자료구조
├─ 단순구조
├─ 선형구조
│  ├─ 리스트
│  ├─ 스택
│  ├─ 큐
│  └─ 데큐
├─ 비선형구조
│  ├─ 트리
│  └─ 그래프
└─ 파일구조
1️⃣ 단순구조 (Primitive Structure)

기본적인 데이터 타입

종류	예
정수	-1, 0, 1
실수	1.2414
문자	a, b, c
문자열	abc
2️⃣ 선형구조 (Linear Structure)

데이터가 일렬로 나열된 형태의 자료구조

📌 리스트 (List)
1. 선형 리스트 (Linear List)
개념

연속되는 기억장치에 저장되는 자료구조

장점

가장 간단한 자료구조

빠른 접근 속도

효율적인 메모리 공간

단점

삽입, 삭제 시 데이터 이동 필요

2. 연결 리스트 (Linked List)
개념

자료를 임의의 공간에 저장하고 포인터로 연결하는 자료구조

장점

노드의 삽입 / 삭제가 용이

기억공간이 연속적일 필요 없음

단점

포인터 저장 공간 필요

순차 접근만 가능하여 접근 속도 느림

종류

단순 연결 리스트

단순 원형 연결 리스트

이중 연결 리스트

이중 원형 연결 리스트

📌 스택 (Stack)
개념

리스트 한쪽 끝에서만 삽입과 삭제가 이루어지는 자료구조

특징
특징	설명
LIFO	Last In First Out (후입선출)
Overflow	스택이 가득 찬 상태에서 삽입
Underflow	스택이 비어있는 상태에서 삭제
활용 사례

웹 브라우저 뒤로가기

프로그램 실행 취소 (Undo)

인터럽트 발생 시 복귀 주소 저장

후위 표기법 계산

함수 호출 관리

깊이 우선 탐색 (DFS)

📌 큐 (Queue)
개념

한쪽에서 삽입이 이루어지고 다른 한쪽에서 삭제되는 자료구조

특징
특징	설명
FIFO	First In First Out (선입선출)
Front	삭제 위치
Rear	삽입 위치
활용 사례

프린터 대기열

은행 창구 대기

콜센터 고객 대기

운영체제 작업 스케줄링

너비 우선 탐색 (BFS)

📌 데큐 (Deque)
개념

양쪽 끝에서 삽입과 삭제가 모두 가능한 자료구조

특징

Double Ended Queue

Stack과 Queue의 장점 결합

활용 사례

우선순위 스케줄링

문서 편집 Undo / Redo

종류
종류	설명
입력 제한 데큐	삽입은 한쪽에서만 가능
출력 제한 데큐	삭제는 한쪽에서만 가능
3️⃣ 비선형구조 (Non-Linear Structure)
🌳 트리 (Tree)
개념

노드들이 나무 형태로 계층적으로 연결된 자료구조

활용 예

가족 계보

연산 수식

회사 조직 구조

Heap 구조

트리 용어
용어	설명
Node	데이터를 저장하는 기본 요소
Root Node	트리의 최상위 노드
Degree	노드의 자식 수
Leaf Node	자식이 없는 노드
Depth	트리의 최대 깊이
Tree Degree	노드 차수 중 최대값
🌲 이진 트리 (Binary Tree)
개념

각 노드가 최대 2개의 자식 노드를 가지는 트리

종류
종류	설명
Full Binary Tree	모든 노드가 자식 0개 또는 2개
Complete Binary Tree	마지막 깊이를 제외한 모든 레벨이 채워짐
📊 그래프 (Graph)

정점(Vertex)과 간선(Edge)으로 이루어진 자료구조

📁 파일구조 (File Structure)

순차 파일 (Sequential File)

색인 파일 (Indexed File)

직접 파일 (Direct File)

⚡ 자료구조의 활용
활용	설명
정렬 (Sorting)	데이터를 일정한 순서로 정렬
탐색 (Searching)	원하는 데이터 찾기
파일 편성 (File Organization)	데이터 저장 방식
인덱스 (Index)	빠른 데이터 검색
🔗 자료구조와 알고리즘

자료구조와 알고리즘은 서로 밀접한 관계를 가지며
자료구조에 따라 알고리즘의 효율성이 달라진다.

⏱ 시간복잡도와 공간복잡도
좋은 알고리즘의 평가 기준
항목	설명
시간 복잡도	알고리즘 실행 시간
공간 복잡도	알고리즘 메모리 사용량

일반적으로 데이터 수 n에 대한 연산 횟수로 표현한다.

복잡도 표기법
표기법	설명
Big-O	최악의 경우
Big-Ω	최선의 경우
Big-θ	평균의 경우
