# 자료구조 (Data Structure)

## 개념
- 컴퓨터에서 처리할 데이터를 효율적으로 관리하고 구조화시키기 위한 방법

---

## 분류

### 단순구조
- 정수 (예 : -1, 0, 1)
- 실수 (예 : 1.2414)
- 문자 (예 : a, b, c)
- 문자열 (예 : abc)

### 선형구조

#### 리스트 (List)

##### 1. 선형 리스트 (Linear List)

###### 개념 
- 연속되는 기억장치에 저장되는 자료구조

###### 장점
- 가장 간단한 자료구조
- 빠른 접근속도
- 효율적인 메모리 공간

###### 단점
- 삽입, 삭제 시 데이터 이동으로 인한 번거로운 작업 필요

##### 2. 연결 리스트 (Linked List)

###### 개념 
- 자료를 임의의 공간에 기억시키고, 순서에 따라 포인터로 연결시킨 자료구조

###### 장점
- 노드의 삽입, 제거가 용이
- 기억공간이 연속적일 필요 없음

###### 단점
- 포인터 부분이 필요하여 선형리스트 대비 낮은 메모리 이용 효율
- 연결을 통해 순차적으로 접근해야 하므로 느린 접근 속도

###### 종류
- 단순 연결 리스트
- 단순 원형 연결 리스트
- 이중 연결 리스트
- 이중 원형 연결 리스트

#### 스택

##### 개념
- 리스트 한쪽 끝으로만 자료의 삽입, 삭제가 이루어지는 자료구조

##### 특징
- 가장 나중에 입력된 데이터가 먼저 삭제되는 후입선출 구조
- 스택에 모든 메모리가 채워져 있는 상태에서 자료를 삽입하려고 하면 Overflow가 발생
- 스택에 모든 메모리가 비워져 있는 상태에서 자료를 제거하려고 하면 Underflow가 발생
 
##### 활용사례
- 웹 브라우저 방문기록 뒤로가기
- 프로그램의 실행 취소 (Undo)
- 인터럽트가 발생하여 복귀주소 저장 시
- 후위 표기법으로 수식을 표현
- 함수 호출 순서 제어
- 깊이 우선 탐색(DFS) 구현

#### 큐

##### 개념
- 한쪽에서 데이터의 삽입이 이루어지고 다른 한쪽에서 데이터의 삭제가 이루어지는 자료구조

##### 특징
- 가장 먼저 입력된 데이터가 먼저 삭제되는 선입선출 구조
- 프런트(F. Front) 포인터 :  가장 먼저 입력된 데이터의 기억공간을 가리키며, 삭제 작업을 할 때 사용
- 리어(R. Rear) 포인터 : 가장 마지막에 입력된 데이터의 기억공간을 가리키며, 삽입 작업을 할 때 사용

##### 활용사례
- 우선순위가 같은 작업 예약(프린터 인쇄)
- 은행 창구 업무의 서비스 순서 대기 처리
- 콜센터 고객 대기시간
- 운영 체제의 작업 스케즐링
- 너비 우선 탐색(BFS) 구현

#### 데큐

##### 개념
- 삽입과 삭제가 양쪽 끝에서 모두 발생할 수 있는 자료구조

##### 특징
- Double Ended Queue의 약자
- Stack과 Queue의 장점만을 활용하여 구성

##### 활용사례
- 우선순위를 조절하는 스케즐링
- 문서 편집의 Undo/Redo

##### 종류
- 입력제한데큐 (Scroll)
- 출력제한데큐 (Shelf)

### 비선형구조

#### 트리

##### 개념
- 노드들이 나무 가지처럼 연결된 계층적 자료구조

##### 예
- 가족 계보, 연산 수식, 회사 조직 구조도, Heap 포현하기 적합

##### 용어 정리
- 노드(Node) : 자료 항목을 가지는 트리를 구성하는 기본 요소
- 근 노드(Root Node) : 트리의 가장 맨 상단에 있는 노드
- 차수(Degree) : 각 노드에서 뻗어나온 가지 수
- 잎사귀 노드(Leaf Node) : 자식이 하나도 없는 노드 (차수가 0인 노드)
- 트리의 깊이(Depth) : 트리에서의 최대 깊이
- 트리의 차수(Degree) : 노드들의 차수 중 최대 차수


#### 그래프

### 파일구조
- 순차 파일
- 색인 파일
- 직접 파일

## 자료구조의 활용

| 활용 | 설명 |
|-----|-----|
| 정렬 (Sorting) | 데이터를 일정한 순서로 나열 |
| 탐색 (Searching) | 원하는 데이터를 찾는 과정 |
| 파일 편성 (File Organization) | 데이터를 저장할 때의 파일 구조 |
| 인덱스 (Index) | 특정 데이터를 빠르게 찾기 위한 구조 |

## 자료구조와 알고리즘의 관계
- 자료구조에 따라 알고리즘이 달라지며, 자료구조와 알고리즘은 서로 의존적인 관계

---

## 시간복잡도와 공간복잡도

### 좋은 알고리즘을 평가하는 요소
- 시간복잡도 : 속도 (일반적으로 메모리 사용량보다 실행속도에 관심)
- 공간복잡도 : 메모리 사용량

### 비교 방법
- 일반적으로 데이터 수 n에 대하여 덧셈, 뺄셈, 곱셈 등의 횟수를 시간복잡도로 표현

| 표기법 | 설명 | 특징 |
|---------|-----|:---:|
| 빅오 (Big-O) | 최악의 경우에 대한 성능을 판단 | 상수항 무시, 영향력 없는 항 무시 |
| 빅오메가 (Big-Ω) | 최선의 경우에 대한 성능을 판단 | X |
| 빅세타 (Big-θ) | 평균의 경우에 대한 성능을 판단 | X |
