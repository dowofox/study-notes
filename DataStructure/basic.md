# 자료구조 (Data Structure)

## 개념
- 컴퓨터에서 처리할 데이터를 효율적으로 관리하고 구조화시키기 위한 방법

---

## 분류

### 단순구조
- 정수 (예 : -1, 0, 1)
- 실수 (예 : 1.2414)
- 문자 (예 : a, b, c)
- 문자열 (예 : abc)

### 선형구조
- 리스트
- 스택
- 큐
- 데큐

### 비선형구조
- 트리
- 그래프

### 파일구조
- 순차 파일
- 색인 파일
- 직접 파일

---

## 자료구조의 활용

| 활용 | 설명 |
|-----|-----|
| 정렬 (Sorting) | 데이터를 일정한 순서로 나열 |
| 탐색 (Searching) | 원하는 데이터를 찾는 과정 |
| 파일 편성 (File Organization) | 데이터를 저장할 때의 파일 구조 |
| 인덱스 (Index) | 특정 데이터를 빠르게 찾기 위한 구조 |

---

## 자료구조와 알고리즘의 관계
- 자료구조에 따라 알고리즘이 달라지며, 자료구조와 알고리즘은 서로 의존적인 관계

---

## 시간복잡도와 공간복잡도

### 좋은 알고리즘을 평가하는 요소
- 시간복잡도 : 속도 (일반적으로 메모리 사용량보다 실행속도에 관심)
- 공간복잡도 : 메모리 사용량

### 비교 방법
- 일반적으로 데이터 수 n에 대하여 덧셈, 뺄셈, 곱셈 등의 횟수를 시간복잡도로 표현

| 표기법 | 설명 | 특징 |
|---------|-----|
| 빅오 (Big-O) | 최악의 경우에 대한 성능을 판단 | 상수항 무시, 영향력 없는 항 무시 |
| 빅오메가 (Big-Ω) | 최선의 경우에 대한 성능을 판단 |---|
| 빅세타 (Big-θ) | 평균의 경우에 대한 성능을 판단 |---|
